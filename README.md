# BayramŞekeri Projesi

Bu proje Git ve GitHub eğitiminde kullanılmak üzere oluşturulmuştur.

## Amaç

Öğrencilerin aşağıdaki konuları pratik yaparak öğrenmesi amaçlanmaktadır:

- Git ile versiyon kontrol
- Branch oluşturma
- Pull Request açma
- Code Review süreci
- Issue Yönetimi 
- GitHub Project
- GitHub Kanban

## Ödev

1. Bu projeye ek olarak her bir proje için  GitHub Projects'de proje oluşturun
2. Bu projelerde To Do - In Development  - In Progess - Test - Done bölümleri açın.
3. Projeleriniz  için  örnekteki gibi issue oluşturup ekip arkadaşlarınıza atama yapın.
4. Her bir issue için yeni bir branch açarak geliştirme yapın.
5. Proje ayarlarından main ve develop branch e pushu kapatın.
6. Beş (5) adet approve olmadan merge edilmeme kuralı ekleyin.
7. PR için temp oluşturun.
8. Branch geliştirmeleri tamamlanıdğında push edip pr açın.  Push yaparken issue id'sini ilk commite mesajınıza eklemeyi unutmayın.
9. Prlara yorum yaparak code review yapın. Approve verin
10. Bir issue üzerinden pr açıldı ise durumu otomatik test çekilsin.
11. PR merge edilirse issue otomatik closed olsun.
